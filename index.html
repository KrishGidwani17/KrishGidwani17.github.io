What Makes Up an Elevator

Common Features of an Elevator Pannel
  • Buttons to select floors
  • Open and close buttons
  • Display saying what floor you are on
  • Arrows signifying going up or going down
  • Emergency buttons
Rare Features of an Elevator Pannel
  • Information about today like date or time
  • Touch screen interface

Overall, these features do make doing everyday features in an elevator very simple to use if they are implemented correctly

Common Sequence of Actions When Using an Elevator
  1. User walks into elevator
  2. User scans through buttons to select floor, and chooses desired floor
  3. User waits for doors to close, or presses a "close door" button
  4. User gets to desired floor and walks out
How Does an Elevator Support the User?

The elevator usually supports the user by providing icons for what each button does, these icons are usually next to the button or on top of the button. Usually, the use of an elevator is very intuitive and if the panel is designed well this is a quick process to figure out how an elevator works.

How Does an Elevator Provide Feedback to the User:
  • When a button is pressed the button lights up signifying a push
  • When doors open or close there is audible “ding”
  • As you pass floors, if there is a display the display manually changes based on what floor you are passing
Common Mistakes with Current Elevators:
  • Mistakenly selecting the wrong floor due to button layout
  • If there are multiple stops for an elevator, some people get off at the wrong floor as there is no information to tell them what floor you have arrived on
Improvements to Current Design:
  • Touch screen display allowing user to input what floor they would like to go to
  • Modern design with minimal colors
  • Colored button for highlighted features -> red for emergency buttons, green for confirming actions
  • Display giving information about time, and date
  • Display those changes based on what floor the user is on
